# 1/6/12/24時間後 降水確率・気温予測NN 学習パイプライン設計

## 背景・目的

現状のダッシュボードは観測値（現在値・直近推移）の可視化のみ。1/6/12/24時間後の降水確率と気温を予測するNNを学習し、将来的にダッシュボードへの予測表示につなげる土台を作る。

今回のスコープは**学習パイプラインのみ**。`server.py`/`client.py`への推論組み込みは別タスク。学習に使うデータは現在の`observations.csv`/`observations_amedas.csv`（2025-09-01〜2025-09-15, 15日分）を使うが、将来データが追加・差し替えされても同じ読み込みロジックで動くように設計する。

## モデル設計

### 方針: 地点共通の単一モデル

地点ごとに個別モデルを作ると、地点あたりのサンプル数が少なく過学習しやすい。単一の共通モデルに「地点の個性」（緯度経度・標高）と「季節・時刻」（周期エンコーディング）を入力として与えることで、沖縄と北海道・夏と冬の違いをモデル自身に学習させる。

### アーキテクチャ

PyTorchのMLP（フラット化した24時間分の時系列 + 静的特徴量を1本のベクトルにして全結合層に通す）。

- 入力
  - 時系列特徴量（直近24時間分、地点あたり8種 × 24時間 = 192次元）: `temp, precip, humidity, solar, sunshine, wind_speed, vapor_pressure, dew_point`
    - `wind_dir`, `cloud`はカテゴリ値のため今回のv1では除外（数値エンコーディングは別途検討）
    - 欠測値は特徴量ごとの学習データ平均で補完（欠測フラグ列は追加しない）
  - 静的特徴量: `lat, lon, elev`（3次元）
  - 時刻の周期エンコーディング: 月・時刻をそれぞれ`sin`/`cos`変換（4次元）
  - 合計 約199次元
- 隠れ層: 全結合2層程度（例: 199→128→64、ReLU、dropout軽め）。パラメータ数はサンプル数（既存データで官署のみ約47,000、アメダス込みで約400,000）に対して過大にならない範囲に収める
- 出力ヘッド: 2つ
  - 降水確率ヘッド: `[1h, 6h, 12h, 24h]`後それぞれの降水確率（シグモイド、4次元）
  - 気温ヘッド: `[1h, 6h, 12h, 24h]`後それぞれの気温（回帰、4次元）

### 損失関数

`loss = BCE(降水確率ヘッド, 正解ラベル) + MSE(気温ヘッド, 正解気温)`

正解ラベル・正解値の定義:
- 降水確率の正解: 対象時刻の`precip >= 0.1`（既存の`isRaining`と同じ閾値）を1/0に変換
- 気温の正解: 対象時刻の`temp`実測値

両損失は単純合算（重み付けなし、v1）。

## データパイプライン

### サンプル抽出条件

地点×基準時刻の組ごとに、以下を満たす場合のみ学習サンプルとする:
- 基準時刻からさかのぼる24時間分の観測が（欠測補完後に）揃っている
- 基準時刻から1/6/12/24時間後の観測（`precip`, `temp`）が存在する

### train/val分割

日付で分割する（末尾数日をvalとし、それ以前をtrain）。ランダムシャッフルによる分割は同日内の前後データがtrain/val両方に混ざりリークするため使わない。具体的な分割日数はデータ量を見て`train.py`内で調整可能なパラメータにする。

### ファイル構成

- `backend/ml/dataset.py`
  - CSV読み込み（既存の`observations.csv`/`observations_amedas.csv`と同じスキーマを想定）
  - 地点×基準時刻ごとのサンプル抽出、特徴量ベクトル化、正解ラベル生成
  - 欠測補完（学習データ平均）
  - train/val分割（日付ベース）
- `backend/ml/model.py`
  - MLPモデル定義（PyTorch `nn.Module`）
- `backend/ml/train.py`
  - 学習ループ、損失計算、検証、モデル保存（`backend/ml/model.pt`）
  - `if __name__ == "__main__"`で実行可能。学習曲線（train/val loss）を標準出力にログ
- `requirements.txt`に`torch`, `pandas`, `numpy`を追加

## 自己チェック

`train.py`を実際の`observations.csv`（官署15日分）に対して数epoch実行し、以下をassertする最小限のスモークテストを`backend/ml/train.py`の`if __name__ == "__main__"`内、または`backend/ml/test_train.py`に用意する:
- 学習が例外なく完走すること
- train lossが1epoch目より最終epochで下がっていること（学習ループが機能している最低限の確認。予測精度そのものの保証はしない）
